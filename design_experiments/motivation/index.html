Motivation — Design Experiments

Every motivator has two faces: the challenge (a CTA that says "here is something worth doing") and the accomplishment (the moment of payoff that says "you did it"). Each mockup below labels both, plus the eligibility condition — when the challenge should actually appear. A "master 50 words" challenge shown to a user who has only seen 12 is demoralizing; the same challenge shown when they're at 38/50 is irresistible.

The 50-words milestone is the canonical case — it runs as a thread through #1 → #4 → #6 so you can trace the challenge → progress → payoff arc end-to-end.

green ring marks the motivation surface in each mockup

What's not yet mocked: the forgetting-curve calendar, voice diary, memory map, study-buddy duo cards, reviewer-note inbox, share-a-sentence flow, native-artifact unlocks. Each follows the same two-element pattern; add a mockup per idea as it becomes the next-most-likely-to-build.

What this gallery deliberately avoids: any streak counter, any days-in-a-row metric, any chip that decays on absence. The whole point is to reward what you can still remember, not whether you tapped in yesterday.
